## Notification Icon

The bell icon in the top navigation shows:

- **Badge with count**: Number of unread notifications
- **Real-time updates**: Counter increases as new notifications arrive
- **Quick access**: Click to view all notifications

<Note>
  The notification counter updates in real-time via WebSocket connections, so you see new alerts instantly.
</Note>

## Notification List

The notifications page displays all your alerts with:

### Visual Indicators

- **Unread notifications**: Highlighted with a pink/purple border
- **Mention notifications**: Orange @ icon
- **Reply notifications**: Arrow icon
- **Assignment notifications**: User icon
- **Publication notifications**: Image icon
- **Playlist notifications**: Monitor icon

### Notification Content

Each notification shows:

- **Author**: Who triggered the notification
- **Production**: Which production the activity occurred in
- **Entity**: The asset or shot involved
- **Task type**: What department/task type
- **Validation status**: Current status (if changed)
- **Timestamp**: When the activity occurred (relative time)
- **Comment text**: Preview of the comment or reply

```javascript
// Notification filtering parameters
{
  task_type_id: string,      // Filter by task type
  task_status_id: string,    // Filter by task status

}
```

## Notification Details

### Mention Notifications

When someone mentions you with `@yourname`:

- Orange @ icon indicates mention
- Full comment text with your mention highlighted
- Option to reply directly
- Mark as read/unread

### Reply Notifications

When someone replies to your comment:

- Shows the reply text first
- Includes the original comment for context
- Distinguished "Reply" section for clarity

### Assignment Notifications

When you're assigned to a task:

- Shows who assigned you
- Displays task information
- Links directly to the task

### Playlist Ready Notifications

When a playlist finishes building:

- Shows playlist name
- Links to view the playlist
- Indicates who created the playlist

## Managing Notifications

### Mark as Read/Unread

Each notification has a toggle to:

- Mark individual notifications as read
- Mark individual notifications as unread
- Toggle updates the counter automatically

### Mark All as Read

Click "Mark all as read" to:

- Clear all unread notifications at once
- Reset the notification counter to zero
- Only available when viewing all notifications (no filters)

<Warning>
  The "Mark all as read" button only appears when you're viewing all notifications without filters applied.
</Warning>

### Show/Hide Comments

Toggle the "Show comments" button to:

- Expand all notification comment text
- Collapse comments for a compact view
- Preference is saved for your session

## Interacting with Notifications

Click on a notification to:

1. **View task details**: Right panel shows full task information
2. **Read full comments**: See complete conversation history
3. **Access entity**: Navigate to the asset or shot
4. **Take action**: Comment, validate, or update status

<Steps>
  <Step title="Click Notification">
    Click anywhere on the notification card (except the read toggle)
  </Step>
  
  <Step title="View Task Panel">
    Task details load in the right sidebar
  </Step>
  
  <Step title="Take Action">
    Comment, validate, or navigate to the full task page
  </Step>
</Steps>

## Task Watching

Stay informed about tasks by watching them:

- **Auto-watch**: Tasks you comment on are automatically watched
- **Manual watch**: Explicitly watch tasks you want to track
- **Filter by watch status**: See notifications only for watched tasks

<Tip>
  Use the watching filter to distinguish between notifications for your assigned tasks vs. tasks you're following.
</Tip>
